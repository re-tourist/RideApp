## UI / UX 设计 Prompt（专业版，可直接使用）

你是一名资深移动端产品设计师与 UI/UX 专家，请参考 Keep App「我的」页面的整体信息架构与空间层级逻辑，为一款**骑行类 App**重新设计「我的（Profile / Me）」页面。
目标是：**从“工具型个人中心”升级为“个人主页 + 社交身份入口”的复合结构**，强调用户的骑行身份、数据积累与社交连接。

### 一、整体设计原则

- 整体风格：**轻量、克制、卡片化、信息分层清晰**
- 参考 Keep 的「上轻下重」结构：
  - 上部：身份与入口（偏展示）
  - 中部：个人数据与骑行状态（偏总结）
  - 下部：功能列表（偏工具）
- 避免强按钮感，减少“管理后台式”布局
- 所有可点击区域应具备明确的跳转意图（个人主页 / 详情页 / 列表页）

------

### 二、顶部 HUD（功能区）设计 ——【重要】

**顶部不再是单一横向条，而是“功能图标区 + 用户信息区”的纵向错位布局**

#### 1️⃣ 顶部功能图标区（最上方）

- 位置：页面最顶部，贴近状态栏
- 布局：**右对齐或左右分布的小图标**
- 功能建议：
  - 通知 / 消息
  - 设置
  - 扫码 / 骑行设备入口（可选）
- 特点：
  - 图标尺寸小
  - 低视觉权重
  - 不抢占用户身份区域的注意力

> ⚠️ 不出现「编辑资料」按钮
> ⚠️ 不与头像在同一水平线上

------

#### 2️⃣ 用户身份区（HUD 下方，核心视觉区域）

- 头像 **单独占据一行**
- 头像尺寸：中等偏大（强调“这是一个人”而不是一个账号）
- 头像下方信息自上而下排列：
  - 昵称（主标题）
  - 用户 ID / 骑行等级 / 骑行段位（副信息）
  - **个性签名（新增）**
    - 示例：
      -「周末骑行爱好者 🚴‍♂️」
      -「通勤 + 轻度训练」
    - 行数限制：1–2 行，弱强调
- 整个区域 **可点击**
  - 点击进入【完整个人主页】
  - 类似 Keep 的“进入个人主页”体验

------

### 三、社交关系与骑行身份补充（新增）

在用户身份区下方，增加一行 **轻量社交数据入口**

- 横向排列的轻量按钮 / 信息块，例如：
  - 骑友（关注的人）
  - 粉丝 / 被关注
  - 俱乐部
  - 勋章 / 成就

设计要求：

- 不做成传统按钮
- 类似「数据标签」或「弱卡片」
- 点击进入对应列表页

------

### 四、个人数据概览区（中部核心）

采用 Keep 风格的 **“我的数据”卡片**

- 卡片标题：我的骑行 / 我的数据
- 重点展示：
  - 本周 / 本月骑行时长
  - 总里程
  - 骑行次数
- 右侧可辅以简洁柱状 / 条形可视化（非强图表）

点击行为：

- 点击卡片 → 进入「数据详情 / 骑行分析页」

------

### 五、功能入口区（下半部分，工具属性）

使用**纵向列表 + 图标**形式，弱化管理感，突出“我与骑行的关系”。

推荐分组顺序：

1️⃣ 与骑行行为强相关

- 骑行记录
- 运动日历
- 我的活动

2️⃣ 与社交和内容相关

- 我的动态
- 骑友
- 收藏 / 加油

3️⃣ 设置与系统

- 系统设置
- 关于 / 帮助

设计要求：

- 图标简洁、风格统一
- 每一项都是“去看”的入口，而不是“去配置”

------

### 六、整体体验目标总结（设计意图）

- 用户一眼看到的不是“你能改什么”，而是：

  - **“你是谁”**
  - **“你骑了多少”**
  - **“你和谁在一起骑”**

- 「我的」页从：

  > 设置中心 → 个人主页入口

- 与 Keep 保持一致的**心理模型**，但内容完全服务于骑行场景
